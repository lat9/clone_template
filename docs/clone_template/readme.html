What it does

With the release of Zen Cart 1.5.5a and its built-in responsive template, I found that creating "clones" of that template by hand were getting to be a pain! Rather than continue with that pain, this plugin came into being! Once you're copied the plugin's files to your store's file-system, you'll find a new admin-menu entry: Tools->Clone a Template

Clone a Template enables you to create a copy of any of your store's templates with a single button-click. The processing copies all template-override files from a source template to a target, copies that source-template's Layout Boxes Controller database settings to the new template and finally updates the target-template's /includes/templates/target_template/template_info.php to identify the source-template's folder-name and the date/time of the cloning operation.

This section identifies the procedure you'll use to install/update or remove the plugin for your Zen Cart.

  • Installation and Updates
  • Uninstall

There are no core-file overwrites required by Clone a Template. Just rename the YOUR_ADMIN folder to match your Zen Cart admin's folder name and then copy the plugin's files to your store's file-system:

  1. /YOUR_ADMIN/clone_template.php
  2. /YOUR_ADMIN/includes/auto_loaders/config.clone_template.php
  3. /YOUR_ADMIN/includes/classes/class.admin.cloneTemplate.php
  4. /YOUR_ADMIN/includes/extra_datafiles/clone_template_filenames.php
  5. /YOUR_ADMIN/includes/init_includes/init_clone_template.php
  6. /YOUR_ADMIN/includes/languages/english/clone_template.php
  7. /YOUR_ADMIN/includes/languages/english/extra_definitions/clone_template_name.php

First, delete the files that were copied to your Zen Cart installation's admin directories. Then use your admin's Tools->Run SQL Patches to run the file /docs/clone_template/uninstall/clone_template_uninstall.sql.
