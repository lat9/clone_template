What it does

With the release of Zen Cart 1.5.5a and its built-in responsive template, I found that creating "clones" of that template by hand were getting to be a pain! Rather than continue with that pain, this plugin came into being! Once you're copied the plugin's files to your store's file-system and enabled the plugin, you'll find a new admin-menu entry: Tools :: Clone a Template

Clone a Template enables you to create a copy of any of your store's templates with a single button-click. The processing copies all template-override files from a source template to a target, copies that source-template's Layout Boxes Controller database settings to the new template and finally updates the target-template's /includes/templates/target_template/template_info.php to identify the source-template's folder-name and the date/time of the cloning operation.

v1.2.0 of Clone a Template introduces the capability to also remove a template's template-override files … for any add-on or cloned templates in your store. This action deletes all files and associated directories and removes all Layout Boxes Controller settings from your database.

Starting with v2.0.0, Clone a Template is now a Zen Cart encapsulated plugin.

This section identifies the procedure you'll use to install/update or remove the plugin for your Zen Cart.

  • Installation and Updates
  • Uninstall

There are no core-file overwrites required by Clone a Template. Just copy the plugin's files to your store's file-system:

  1. /zc_plugins/CloneTemplate/version-num/manifest.php
  2. /zc_plugins/CloneTemplate/version-num/admin/clone_template.php
  3. /zc_plugins/CloneTemplate/version-num/admin/includes/classes/class.admin.cloneTemplate.php
  4. /zc_plugins/CloneTemplate/version-num/admin/includes/extra_datafiles/clone_template_filenames.php
  5. /zc_plugins/CloneTemplate/version-num/admin/includes/languages/english/lang.clone_template.php
  6. /zc_plugins/CloneTemplate/version-num/admin/includes/languages/english/extra_definitions/lang.clone_template_name.php
  7. /zc_plugins/CloneTemplate/version-num/Installer/ScriptedInstaller.php

Note: The version-num refers to the plugin's current version, e.g. v2.0.0.

Once the files are copied, navigate to your admin's Modules :: Plugins and Enable the Clone a Template plugin.

Navigate to the admin's Modules :: Plugins, and Un-Install the Clone a Template plugin. To fully remove the plugin from your site, you'll also need to delete the /zc_plugins/CloneTemplate sub-directory (and all its files).
